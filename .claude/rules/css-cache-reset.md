# HTML/CSS 変更時のデザイン整合性 & ブラウザキャッシュリセットルール

頭痛疾患 HTML シリーズを編集するにあたり、デザインシステムの整合性維持と、変更内容を正しくブラウザで動作確認するための手順を定義する。

## デザイン整合性のルール

本リポジトリで作成・編集するすべての HTML ページは、デザインシステムの一貫性を守る必要がある。

### 1. 権威ソースの継承
- すべてのデザイン（CSS 変数、コンポーネントクラスなど）は、`Types-of-headache/html-files/Headaches/Migraine.html` を基準とする。
- 共通で使用する以下の要素は変更してはならない：
  - CSS 変数群 (`--navy`, `--blue`, `--teal`, `--orange`, `--red`, `--green`, `--purple` など)
  - 共通コンポーネントクラス (`.card`, `.alert`, `.tbl`, `.mmd`, エビデンスバッジなど)

### 2. ページ固有デザインとクラス名のプレフィックス化
- 疾患ごとにヒーローグラジエントなどの個別色を使用するが、これ以外の固有スタイルを追加する場合は、他のページに影響を与えないよう注意する。
- 固有 CSS クラスを定義する場合は、名前衝突を防ぐため、必ずページ固有のプレフィックス（例: SGB なら `.sgb-xxx`）を付与して安全にカプセル化する。

## ブラウザ検証時のキャッシュリセット（必須）

HTML やスタイルシートを変更した後、ローカルのブラウザ（または Playwright 等）で動作検証する際、ブラウザのキャッシュによって古い CSS 定義や Mermaid レイアウトが残ることで、表示崩れや動作不具合が誤検知される場合がある。

### キャッシュリセット手順

編集した HTML をブラウザで確認する際は、必ずキャッシュを無効化した状態でリロード（ハード再読み込み）を実行すること。

1. **開発者ツール (DevTools) のキャッシュ無効化設定（推奨）**
   - Chrome / Edge 等のブラウザで `F12`（または `Cmd+Option+I`）キーを押し、開発者ツールを開く。
   - 「Network」タブに移動し、**「Disable cache (キャッシュを無効化)」**にチェックを入れる。
   - 開発者ツールを開いている間は、通常のページリロードでも常に最新の CSS が読み込まれる。

2. **ハード再読み込みのショートカット**
   - 開発者ツールが開いていない場合でも、以下のショートカットを実行してキャッシュを強制クリアしてリロードする：
     - **macOS (Chrome/Firefox/Safari)**: `Cmd + Shift + R` または `Shift + リロードボタンクリック`
     - **Windows/Linux (Chrome/Firefox/Edge)**: `Ctrl + F5` または `Ctrl + Shift + R`

3. **ローカル確認時の注意**
   - テキストのカラー変更や Mermaid ダイアグラム内の配置調整を行った後は、必ず上記の手順でキャッシュをリセットし、変更が正しくブラウザ上にレンダリングされることを目視で確認すること。
