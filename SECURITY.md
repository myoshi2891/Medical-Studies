# セキュリティ方針 / Security Policy

本リポジトリは頭痛疾患の医療**教育**コンテンツと、患者報告アウトカム（PROM）を計算する完全クライアント型 Web アプリ（`web-next/`）を含む。詳細な設計・是正案は [`docs/publishing/04-security-policy.md`](docs/publishing/04-security-policy.md) を参照。

## 脆弱性の報告

脆弱性を発見した場合は、**公開 Issue を作成せず**、GitHub の **Private Vulnerability Reporting**（リポジトリの "Security" タブ →「Report a vulnerability」）から非公開で報告してください。

- 報告に含めてほしい情報: 再現手順、影響範囲、対象コミット/URL、想定される深刻度。
- **PoC に実在の患者データ・個人情報を含めないでください。**
- 対応目安: 受領後、内容を確認し、修正方針を検討します（ベストエフォート。本プロジェクトは教育目的の個人／小規模運用を想定）。

## 対象範囲

このアプリはサーバサイド API・サーバ秘密情報を持たない（`web-next/app/api/` は存在しない）。したがって主な対象は以下:

- **対象**: クライアントサイドの XSS、依存パッケージの既知脆弱性、セキュリティヘッダ（CSP 等）の欠如、OAuth トークンの取り扱い、`localStorage` に保存される健康データの保護。
- **対象外（設計上該当しない）**: サーバ侵害、サーバ側データベース、バックエンド認証。

## データの取り扱い

- 頭痛日誌・PROM スコアは利用者の**端末内 `localStorage`** にのみ保存され、運営側サーバへ送信・保存されません。
- Google 連携（任意機能）の OAuth トークンはブラウザメモリ上のみで保持し、スコープは利用者自身が本アプリで作成したファイルに限定（`drive.file`）されます。
- 共有端末では健康データが端末に残りうる点に注意してください（消去導線・注意喚起は改善中）。

## 免責

本アプリが提供するスコア・情報は**教育・参考目的**であり、医学的診断・治療の助言ではありません。詳細は [`docs/publishing/05-legal-and-regulatory.md`](docs/publishing/05-legal-and-regulatory.md) を参照してください。
