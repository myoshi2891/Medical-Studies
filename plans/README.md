# 頭痛専門プラットフォーム拡張計画（web-next）

`.agents/skills/improve` スキルに基づき、2026-07-09 に作成した実装計画群。対象は `web-next/`
（Next.js App Router アプリ）。**本ディレクトリは計画文書のみ**であり、コード変更は含まない。
各計画は、この会話・監査を見ていないゼロコンテキストの実行者が単独で読めるよう自己完結で書かれている。

各実行者へ: 着手前に該当プランを最後まで読み、STOP 条件を守り、完了時に自分の行の Status を更新すること。
プラン群は 2 系統ある: **コンテンツ戦略系 001〜007**（Planned at `a9e470c`）と、
**公開ガバナンス系 008〜015**（`docs/publishing/` の監査所見 F1〜F7 由来。Planned at `6614b7c`）。

## 全体像（4 本柱）

プラットフォームは 4 本柱で構成する（詳細は 001）: **情報収集 / 研究 / 治療 / 療養**。
横串として **カテゴリ体系**（002）と **技術基盤**（007）が全柱を支える。
不変の原則: 教育専用・診断補助なし・薬機法 SaMD 非該当を維持する（001）。

## 依存関係

- **001 が全プランの前提**（ビジョン・不変の原則・現状分析）。最初に通読する。
- **002（カテゴリ体系）が 004・005・006 の前提**: ルート slug（`/treatment/*`・`/research/*`・
  `/care/*`）を 002 が確定し、コンテンツプランはそれを使用する。
- **003（情報収集運用）が 004・005・006 の前提**: エビデンス表記・参考文献・鮮度メタの規約を
  003 が定め、各コンテンツはそれに従う。
- **007（技術基盤）は 002 に依存**（カテゴリ体系をレジストリ化）。コンテンツが 10 ページ超
  増える前に着手するのが望ましい。003 の鮮度メタを実装で受ける。
- コンテンツ 3 プラン（004/005/006）は相互独立。P1 の 004 を先行し、005/006 は並行可能。
- **008 / 010 は 2 フェーズ構成**: Phase A（メタデータの器）は即実行可、Phase B（レダクション／
  公開除外）は権利者・出典の確認結果という決定ゲート成立時のみ。
- **014（CI）は 009 に依存**: ライセンスゲートのツール・failOn リストを 009 の棚卸し結果に合わせる
  （009 未実施でも既定リストで実行可）。
- **015（履歴書き換え）は必ず最後**: 全 SHA が変わり、他プランの drift check 基準
  （`a9e470c`・`6614b7c` 等）を無効化するため。実行後に 015 Step 7 で参照 SHA を更新する。
- ガバナンス系（008〜015）とコンテンツ系（001〜007）は独立して進められる。ただし公開リスクの
  低減が目的である以上、P0 の 008 Phase A・009 はコンテンツ拡張より先に済ませることを推奨する。

## 推奨着手順

1. **001 → 002 → 003**（設計・規約の土台。順に確定）
2. **007 の Step 0〜1**（コンテンツレジストリの器を先に用意すると、以降のページ追加で
   登録漏れを機械検知できる）
3. **004**（P1・治療。最も需要が高い薬物療法の P1 ページ 2 種を先行）
4. **006 → 005**（療養は既存 PROM 資産と結ぶだけで価値が出る／研究は継続運用が要るため後）
5. **007 の残り Step**（ナビ再設計・サイトマップ・用語集展開をコンテンツ増加に合わせて）

## 見送り事項（considered and rejected / 再監査不要）

- **DICOM 読込・患者固有 3D 再構築・AI 診断**: `docs/architecture.md` v1.3 で恒久的に非対象と
  宣言済み。SaMD 該当リスクのため採用しない。
- **診断アルゴリズム・治療意思決定支援・用量計算機/相互作用チェッカー**: 医療機器（SaMD）該当
  リスク。教育専用スコープを外れるため採用しない（001 原則 1、004 STOP 条件）。
- **既存 URL のリネーム/再編**: 外部リンク切れを招く。カテゴリ体系は追加のみで成立する設計（002）。
- **フォーラム/ユーザー投稿機能**: 医療情報の品質管理が不能になるためビジョン外（002）。
- **多言語化・認証・収益化・検索 SaaS 導入**: 現段階では優先度管理のみ（007 の将来スパイク）。
  必要性が実証されるまで着手しない。
