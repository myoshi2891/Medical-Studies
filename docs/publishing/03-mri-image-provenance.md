## 2. 匿名化と著作権は別問題

> [!IMPORTANT]
> PHI 匿名化（患者を特定できない状態にすること）と、画像の**著作権・利用許諾**（誰がその画像を公開・再配布してよいか）は**別個の要件**である。
> 匿名化済みであっても、画像の著作権者が公開を許諾していなければ再配布はできない。医用画像は撮影施設・機器メーカー・データセット提供元などに権利が及びうる。

## 3. 出典記録テンプレート（要記入）

各シリーズについて以下を確定し、`manifest.json` の各 series `provenance` に記録する。**確定済み（開発者本人保有）。**

| 項目 | brain（8 枚） | cervical（8 枚） |
|---|---|---|
| 元データの取得元（施設／データセット名） | 自己保有（本人撮影分） | 自己保有（本人撮影分） |
| 著作権者 | リポジトリ管理者（本人保有） | リポジトリ管理者（本人保有） |
| 利用許諾（教育公開・再配布の可否） | 可（本人保有・`permission: "own"`） | 可（本人保有・`permission: "own"`） |
| ライセンス（該当する場合） | 該当なし | 該当なし |
| 帰属表示の要否 | 不要 | 不要 |
| 撮影種別 | 頭部 T2 軸位（代表スライス） | 頚椎 矢状断（代表スライス） |

### 想定される取得元の類型（確認の指針）

- 自施設・自身で撮影／権利を保有 → 公開許諾を自ら付与できる（その旨を明記）。
- 公開教育用データセット（例: オープンライセンスの医用画像コレクション） → データセットのライセンスと帰属義務に従う。
- 第三者施設・商用データ → 明示的許諾がなければ公開不可。

## 4. 公開継続の可否判断

- 出典・許諾が確認できるまで、公開継続は権利上のリスクを伴う。
- 判断オプション:
  1. 出典・許諾を確認し、`manifest.json` と本文書に記録したうえで公開継続。
  2. 確認が取れるまで当該画像を公開対象から外す（実装は別プラン。本文書では方針のみ）。
  3. 権利の明確なオープンライセンス画像へ差し替える。
